## 🚀 Usage

`~/.finjuice` is the default data directory. This template is for users who want a separate
custom repository and therefore need `--data-dir`.

### Option 1: Using --data-dir flag

```bash
cd ~/path/to/finjuice
finjuice --data-dir ~/Documents/my-finance-data refresh
```

### Option 2: Using environment variable

```bash
export FINJUICE_DATA_DIR=~/Documents/my-finance-data
finjuice refresh
```

### Option 3: Working in data directory

```bash
cd ~/Documents/my-finance-data
finjuice refresh  # Uses current directory
```

## 📝 Workflow

1. **Import XLSX files**: Download from Banksalad and place in `imports/`
   ```bash
   cp ~/Downloads/2024-10-27~2025-10-27.xlsx imports/
   ```

2. **Run pipeline**: Process transactions, apply tags, detect transfers
   ```bash
   finjuice --data-dir ~/Documents/my-finance-data refresh
   ```

3. **Review results**: Check generated reports in `exports/reports/`
   ```bash
   cat exports/reports/monthly_spend.csv
   ```

4. **Commit changes**: Track your transaction history with git
   ```bash
   git add transactions/
   git commit -m "chore: update transactions for 2025-01"
   git push
   ```

## 🔧 Customization

### Edit Tagging Rules

Modify `rules.yaml` to add your own tagging rules:

```yaml
version: 1
rules:
  - name: my_subscription
    match: "Netflix|Spotify"
    fields: [merchant_raw]
    tags: ["구독", "엔터테인먼트"]
    priority: 85
```

### Configuration (Future)

Create `config.toml` for custom settings:

```toml
[pipeline]
default_currency = "KRW"

[tagging]
confidence_threshold = 0.7

[export]
date_format = "%Y-%m-%d"
```

## 🆘 Troubleshooting

### "No such file or directory: data/transactions"

The transactions directory hasn't been created yet. Import an XLSX once, or run the refresh
pipeline against this repository:

```bash
finjuice --data-dir "$PWD" refresh
```

### "No XLSX files found in imports/"

Make sure you've placed Banksalad export files in the `imports/` directory:

```bash
ls imports/  # Should show .xlsx files
```

### "Permission denied" errors

Check that you have write permissions:

```bash
ls -la  # Check ownership
chmod -R u+w .  # Grant write permissions
```

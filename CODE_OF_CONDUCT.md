# Code of Conduct

finjuice is a local-first personal finance project. Community discussion should
stay respectful, privacy-conscious, and focused on helping people work safely
with their own data.

## Expected Behavior

- Be direct and constructive when discussing bugs, design tradeoffs, and support
  requests.
- Assume financial workflows are sensitive even when examples look small.
- Use synthetic, redacted, or generalized examples instead of asking for raw
  transaction rows, account names, exports, screenshots, or private files.
- Respect different levels of Python, CLI, finance, and AI-agent experience.
- Keep issue and pull request conversations scoped to reproducible behavior and
  documented project goals.

## Unacceptable Behavior

- Harassment, insults, discriminatory language, or personal attacks.
- Pressuring someone to disclose private financial data.
- Posting raw Banksalad exports, account numbers, transaction rows, private
  paths, screenshots with financial details, or secrets.
- Sharing another person's private data without explicit permission.
- Repeatedly pushing off-topic product, investment, tax, or legal advice.

## Reporting

For conduct concerns, open a private security advisory if private data or a
security issue is involved. Otherwise, contact the maintainer through the
repository discussion or issue channels and include only the minimum context
needed to understand the concern.

Maintainers may edit or remove content that exposes private financial data,
close conversations that cannot be handled safely, or restrict participation for
repeated violations.

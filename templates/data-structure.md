## CSV Partition Schema

Each `transactions/YYYY/MM/transactions.csv` file contains these columns:

```csv
row_hash,date,time,type_raw,type_norm,major_raw,minor_raw,merchant_raw,memo_raw,
amount,account,currency,counterparty,datetime,category_rule,category_final,
tags_rule,tags_ai,tags_manual,tags_final,confidence,needs_review,is_transfer,
transfer_group_id,file_id,source_row
```

### Key Fields

- **row_hash**: SHA256 hash for deduplication
- **date/time**: Original transaction date/time from Banksalad
- **datetime**: ISO8601 normalized datetime
- **type_norm**: Normalized type (expense|income|transfer|other)
- **amount**: Negative for expenses, positive for income
- **category_final**: Single category used for report aggregation
- **tags_final**: Final merged tags from rules, AI, and manual edits
- **is_transfer**: 1 if identified as internal transfer, else empty
- **transfer_group_id**: Groups paired transfers (e.g., T0001)
- **file_id/source_row**: Compact traceability back to the original import file and row

## Typical File Sizes

| File/Directory | Size (1 year data) | Notes |
|----------------|--------------------|--------------------|
| Single CSV partition | ~30KB | ~200 transactions/month |
| Full `transactions/` | ~400KB | 13 months of data |
| Single import XLSX | ~100KB | Raw Banksalad export |
| Master export | ~300KB | All transactions + formatting |
| Git repository | ~500KB | With 1 year of CSV history |

## Migration from Old Structure

If you have existing `data/` in the program repository:

```bash
# 1. Create new data repository
mkdir ~/Documents/my-finance-data
cd ~/Documents/my-finance-data
git init

# 2. Copy structure
cp <prog-repo>/templates/.gitignore.data .gitignore
cp <prog-repo>/templates/README.data.md README.md
cp <prog-repo>/data/examples/rules.yaml rules.yaml

# 3. Move data
cp -r <prog-repo>/data/transactions .
cp -r <prog-repo>/data/imports .
# (exports can be regenerated, no need to copy)

# 4. Initialize git
git add .gitignore README.md rules.yaml transactions/
git commit -m "init: migrate finance data from program repo"

# 5. Point pipeline to new location
export FINJUICE_DATA_DIR=~/Documents/my-finance-data
finjuice refresh  # Verify it works

# 6. Clean up old data directory
cd <prog-repo>
rm -rf data/transactions data/imports data/exports
```

## Best Practices

### Git Workflow

1. **Track partitions**: CSV files are line-oriented, git-friendly
2. **Ignore raw files**: XLSX files are binary, create large diffs
3. **Commit regularly**: After each import cycle
4. **Use descriptive messages**: "chore: add 2025-01 transactions"

### Security

1. **Private repository**: Never make data repo public
2. **Separate repos**: Keep program code and data in different repositories
3. **Backup regularly**: Push to private GitHub/GitLab
4. **Use SSH keys**: For secure authentication

### Performance

1. **Monthly partitions**: Faster reads (8K tokens vs 170K for full dataset)
2. **Regenerate exports**: Don't commit large master XLSX files
3. **Query partitions directly**: Use finjuice commands or DuckDB-backed reads for complex queries
